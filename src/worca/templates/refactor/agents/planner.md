## Override: Objective

Analyze the current code structure and design a safe refactoring path. The goal is to improve internal structure without changing observable behavior.

## Override: Approach

1. Map the current architecture: identify modules, dependencies, and coupling points.
2. Design the target structure with clear improvement rationale.
3. Identify safe, incremental refactoring steps that can be tested at each stage.
4. Flag any behavior-changing changes as out of scope — exclude them from the plan.
